A type guard that checks if a gived value is defined meaning it neighter `null` nor `undefined`.

## Usage

```tsx
import { isDefined } from '#';
import * as React from 'react';

const Component = () => {
  const maybeValue: string | null | undefined = 'Hello';

  if (isDefined(maybeValue)) {
    // maybeValue is now narrowed to `string`
    console.log(maybeValue); // Output: 'Hello'
  }

  const anotherValue: number | null = null;
  const isAnotherDefined = isDefined(anotherValue); // Output: false

  return null;
};

export default Component;
```

## Type Declaration

```ts
export declare type Defined<T> = Exclude<T, null | undefined>;

export declare function isDefined<T>(value: T): value is Defined<T>;
```

## Reference

| Parameter | Type | Description         |
| --------- | ---- | ------------------- |
| `value`   | `T`  | The value to check. |

### Return

| Type                  | Description                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------ |
| `value is Defined<T>` | `true` if the value is defined, meaning it is neither null nor undefined, otherwise false. |

## Common Pitfalls

### Not using the return value

The `isDefined` function is a type guard. To benefit from its type narrowing capabilities, use it within a conditional statement (e.g., `if (isDefined(value)) { ... }`). The return value itself doesn't provide additional information beyond `true` or `false`.

```ts
const maybeValue: string | null = 'Hello';

// Don't just check the return value
const isDefined = isDefined(maybeValue);

// Correct usage
if (isDefined(maybeValue)) {
  // maybeValue is now narrowed to `string`
  console.log(maybeValue.toUpperCase());
}
```

### Using isDefined with falsy values

While `isDefined` checks for null and undefined, it doesn't consider other falsy values like `0`, `''` (empty string), `NaN`, or `false`. If you need to check for truthiness, use JavaScript's built-in mechanisms like `if (value)` or a more specific type guard.

```ts
const numberValue = 0;

if (isDefined(numberValue)) {
  // This block will execute, even though `numberValue` is falsy
  console.log('Number is defined:', numberValue);
}

// If you want to exclude falsy values, use a direct check
if (numberValue) {
  // This block will not execute since `0` is falsy
  console.log('Number is truthy:', numberValue);
}
```
